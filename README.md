# ZERO — Sistema de Armado de Pedidos

Single-service deployment: **Express API + Next.js frontend** on one port.  
Stack: Next.js 14 · Node/Express · MySQL 8 · Knex · JWT · Railway

---

## Architecture

```
PORT (Railway-assigned)
 │
 ├─ /health            → Express healthcheck
 ├─ /api/*             → Express API routes  (JWT-protected)
 ├─ /_next/static/*    → Next.js hashed assets (1-year cache)
 └─ *                  → Next.js SSR handler (pages + routing)
```

The Express server loads Next.js's standalone output as a request handler — no separate frontend process needed.

## Deploy to Railway

### Step 1 — Create services

1. Go to [railway.app](https://railway.app) → New Project
2. Add **MySQL** plugin (Railway provides host/credentials automatically)
3. Add service from your GitHub repo (this repo root)

### Step 2 — Set environment variables

In Railway → your service → Variables, add:

| Variable | Value |
|---|---|
| `NODE_ENV` | `production` |
| `DB_HOST` | from MySQL plugin (copy `MYSQL_HOST`) |
| `DB_PORT` | `3306` |
| `DB_NAME` | from MySQL plugin |
| `DB_USER` | from MySQL plugin |
| `DB_PASSWORD` | from MySQL plugin |
| `JWT_ACCESS_SECRET` | 64-char random hex (see below) |
| `JWT_REFRESH_SECRET` | 64-char random hex |
| `JWT_ACCESS_EXPIRES` | `15m` |
| `JWT_REFRESH_EXPIRES` | `7d` |
| `LOG_LEVEL` | `info` |
| `INTEGRATION_PROVIDER` | `local` |

Railway injects `PORT` automatically — **do not set it manually**.

Generate JWT secrets:
```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

### Step 3 — Run migrations (once after first deploy)

```bash
# Via Railway CLI
railway run node -e "
  require('dotenv').config();
  const knex = require('knex');
  const cfg = require('./server/db/knexfile.js');
  knex(cfg.production).migrate.latest().then(() => process.exit(0));
"
```

Or use the Railway dashboard shell.

### Step 4 — Done

Railway builds with `npm install && npm run build` and starts with `node server/index.js`.  
Health check: `GET /health`

## Local development

### Option A — Unified (mirrors production exactly)

```bash
# 1. Install all deps
npm install
cd frontend && npm install && cd ..

# 2. Configure env
cp .env.example .env
# Edit .env with local DB creds

# 3. Build frontend once
npm run build

# 4. Start unified server
npm start
# → http://localhost:4000  (API + frontend)
```

### Option B — Split (faster iteration)

```bash
# Terminal 1 — API server (auto-restart on change)
npm run dev:server

# Terminal 2 — Next.js dev server (hot reload)
cd frontend
cp ../.env.example .env.local
# Set: NEXT_PUBLIC_API_URL=http://localhost:4000
npm run dev
# → http://localhost:3000
```

## API endpoints

See [`docs/openapi.yml`](docs/openapi.yml) for full spec.

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/health` | — | Health check (DB ping + uptime) |
| POST | `/api/auth/login` | — | Login → access + refresh tokens |
| POST | `/api/auth/refresh` | — | Rotate tokens |
| POST | `/api/auth/logout` | ✓ | Revoke refresh token |
| GET | `/api/orders` | ✓ | List orders (paginated) |
| GET | `/api/orders/:id` | ✓ | Order detail + items + locations |
| PATCH | `/api/orders/:id/pack` | ✓ operator | Pack order (emits event) |
| POST | `/api/orders/:id/items/:id/missing` | ✓ operator | Mark item missing |
| POST | `/api/orders/:id/items/:id/replace` | ✓ operator | Replace item |
| POST | `/api/orders/:id/supervisor-override` | ✓ admin | Supervisor override |
| GET | `/api/orders/:id/events` | ✓ | Order event log |
| POST | `/api/scans` | ✓ operator | Register barcode scan |
| GET | `/api/kpi/today` | ✓ | Today's summary metrics |
| GET | `/api/kpi/operators` | ✓ | Per-operator efficiency |
| GET | `/api/kpi/errors` | ✓ | Scan error breakdown |
| GET | `/api/kpi/performance` | ✓ | Hourly throughput chart |
| GET | `/api/locations` | ✓ | Warehouse locations |
| POST | `/api/locations` | ✓ admin | Create location |
| POST | `/api/locations/products/:sku/location` | ✓ | Assign SKU → location |

## Tiendanube integration (hooks — not yet implemented)

The codebase has stub hooks ready. To integrate:

1. Set `INTEGRATION_PROVIDER=tiendanube` and add `TIENDANUBE_STORE_ID` + `TIENDANUBE_ACCESS_TOKEN`
2. Implement `fetchOrders()` and `markPacked()` in `server/integrations/tiendanube/client.js`
3. Uncomment the real client in `server/integrations/index.js`
4. Add the `markPacked` call in `server/services/orderService.js → markPacked()`
